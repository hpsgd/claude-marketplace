## Output

**PRD: Bulk User Import**

**Problem statement:** Admins at mid-market accounts (50+ seats) spend 20-40 minutes manually adding team members one at a time during initial setup. This is the top-reported friction point in onboarding feedback (34 tickets in last 6 months) and is associated with a 15% lower day-30 activation rate for accounts >25 users.

**RICE prioritisation:**
- Reach: 2,400 accounts with 25+ seats
- Impact: 3 (massive — directly reduces onboarding time by estimated 80%)
- Confidence: 80% (strong qualitative evidence; no A/B data yet)
- Effort: 3 person-weeks
- RICE Score: 2400 × 3 × 0.80 / 3 = **1,920**

**Target user:** Account admins at companies migrating from another tool or onboarding a new department.

**Success metrics:**

Leading: % of new accounts with 25+ users who attempt CSV import in first 7 days (target: 40% of eligible accounts).

Lagging: Day-30 seat activation rate for accounts using bulk import vs manual (target: ≥85% vs current 70%).

Guardrail: Support tickets related to import errors (must not exceed 5% of import attempts); CSV import error rate (must stay <5%).

**Out of scope:** SSO directory sync (separate project). SCIM provisioning. Editing or removing users via CSV (import-only for v1).

**Risks and pre-mortem:**
- CSV format ambiguity leads to data errors at scale — mitigation: strict validation with clear error messages per row
- Admins import wrong email domain — mitigation: confirmation screen showing first 5 rows before committing
- Large files (1,000+ users) cause timeout — mitigation: async processing with email confirmation

**Rollout strategy:**
- Phase 1: Invite-only beta (20 high-touch accounts, CSM-assisted)
- Phase 2: Enable for all accounts >25 seats
- Phase 3: GA with self-serve documentation

**In scope:** CSV upload via admin panel, validation with per-row error reporting, async processing for files >100 users, email notification on completion.
