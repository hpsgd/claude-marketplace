**Usability Test Plan: New Onboarding Flow**

**Research questions:**

| # | Question | Type | Decision it informs |
|---|---|---|---|
| RQ1 | Can new users complete the onboarding flow without assistance and reach their first project within 10 minutes? | Behavioural | Ship/no-ship decision |
| RQ2 | Where in the flow do users pause, hesitate, or make errors? | Behavioural | Which steps to redesign |
| RQ3 | Do users understand the value of each onboarding step while doing it? | Comprehension | Copy/UX writing improvements |
| RQ4 | What is the perceived difficulty of each step? | Attitudinal | Prioritisation of fixes |

**Methodology:**
- Moderated (exploration — need to understand WHY, not just IF)
- Remote video call
- Think-aloud protocol
- Live product (new onboarding flow in staging)

**Participants:**

Number: 6 (5 catches ~85% of usability issues per Nielsen; 6 for slight buffer given time sensitivity)

| Criterion | Requirement |
|---|---|
| User type | New to Clearpath — no prior account |
| Role | Project manager, team lead, or operations role |
| Tech literacy | Comfortable with web apps; not a developer |
| Company size | 20-500 employees (matches target segment) |
| Exclusions | Clearpath employees, current customers, participants in last 60 days, software developers |

**Screener questions:**

| # | Question | Accept | Reject |
|---|---|---|---|
| S1 | "Do you currently use Clearpath?" | No | Yes |
| S2 | "What is your primary role?" | PM, team lead, ops | Developer, designer |
| S3 | "How often do you use project management software?" | Daily/Weekly | Never |

**Tasks:**

| # | Scenario | Success criteria | Time limit | RQ |
|---|---|---|---|---|
| T1 | "You just started at a new company. They use Clearpath for project tracking. You've been sent an invite — log in and get your workspace set up." | Reaches project creation screen without help | 10 min | RQ1, RQ2 |
| T2 | "Your manager asked you to add two team members to the workspace." | 2 invites sent successfully | 5 min | RQ1, RQ2 |
| T3 | "Create your first project from a template." | Project created with at least one task | 5 min | RQ1, RQ2, RQ3 |

T3 is intentionally the most challenging step — it reveals how the template selection step performs under pressure.

**Moderator guide:**

Introduction: "We're testing the product, not you. There are no wrong answers. Think out loud — tell me what you're seeing, thinking, wondering."

Per task: Read scenario aloud. Observe without intervening. Note: hesitations (>5s), backtracking, verbal confusion.

Intervention policy: Only intervene if participant is stuck for >3 minutes with no progress. Neutral prompt: "What are you looking for right now?"

Follow-up probes (post each task):
- "What did you expect to happen when you [action]?"
- "Was anything confusing or surprising?"
- "On a scale of 1-7, how easy was that task?" (SEQ)

Debrief: "Which task felt hardest? Why? What would you change?"

**Metrics:**

| Metric | Definition | Target | How measured |
|---|---|---|---|
| Task success rate | % completing T1-T3 unassisted | ≥80% | Binary per task |
| Time on task (T1) | Seconds from task start to first project created | <600s (10 min) | Stopwatch |
| Error rate | Wrong actions or backtracking per task | <2 per task | Observer count |
| SEQ | Single Ease Question per task (1-7) | ≥5.5 | Post-task questionnaire |

**Logistics:**

| Item | Detail |
|---|---|
| Tool | Lookback (remote, think-aloud recording) |
| Recording | Screen + audio |
| Consent | Verbal consent + written form emailed before session |
| Session duration | 60 minutes (tasks 20-30 min, intro + debrief 30 min) |
| Schedule | Max 2 sessions/day (avoid moderator fatigue) |
| Pilot session | Run with internal colleague before first real session |
| Deliverable | Findings report with severity ratings within 5 days of final session |
