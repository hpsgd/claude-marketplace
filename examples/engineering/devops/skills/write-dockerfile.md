## Dockerfile

```dockerfile
# Stage 1: Build — install dependencies in isolation (cached until requirements.txt changes)
FROM python:3.13-slim AS build
WORKDIR /app
COPY requirements.txt ./
RUN pip install --no-cache-dir --prefix=/install -r requirements.txt
COPY . .

# Stage 2: Runtime — minimal image, no build tools, non-root user
FROM python:3.13-slim AS runtime
WORKDIR /app

# Create non-root user before copying application files
RUN groupadd --system --gid 1001 appgroup && \
    useradd --system --uid 1001 --gid appgroup appuser

# Copy installed packages and application source
COPY --from=build /install /usr/local
COPY --from=build --chown=appuser:appgroup /app .

USER appuser

ENV PYTHONUNBUFFERED=1
ENV PYTHONDONTWRITEBYTECODE=1

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD ["python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"]

CMD ["uvicorn", "inventory_api.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**Stage strategy:** Two stages. The build stage installs all dependencies into `/install` using `--prefix` to keep them isolated. The runtime stage copies only the installed packages and source — no pip, no build cache, no wheel files. Total runtime image size is roughly 40% smaller than a single-stage equivalent.

**Base image choice:** `python:3.13-slim` over `python:3.13-alpine` because FastAPI's dependencies (pydantic, cryptography) have C extensions that require glibc. Alpine uses musl libc and would require a separate build step for native compilation. If you have no native extensions, `alpine` is a reasonable alternative for a smaller image.
