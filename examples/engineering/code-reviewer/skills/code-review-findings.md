## Prompt

Review this PR. It adds a new endpoint for fetching order history with line items.

```python
# app/routes/orders.py
from fastapi import APIRouter, Query
from app.db import get_session
from app.models import Order, LineItem

router = APIRouter()

@router.get("/orders")
async def get_orders(customer_id: str = Query(...)):
    session = get_session()

    # Fetch orders for customer
    query = f"SELECT * FROM orders WHERE customer_id = '{customer_id}'"
    orders = session.execute(query).fetchall()

    results = []
    for order in orders:
        # Get line items for each order
        items = session.query(LineItem).filter(LineItem.order_id == order.id).all()

        total = sum(item.unit_price * item.quantity for item in items)
        results.append({
            "order_id": order.id,
            "status": order.status.upper(),
            "total": total,
            "items": [{"sku": i.sku, "qty": i.quantity} for i in items],
        })

    return {"orders": results}
```

**Correctness — AttributeError on null status** `app/routes/orders.py:34`

`order.status.upper()` will raise `AttributeError` if `status` is `NULL` in the database. No guard is present.

**Evidence:** `"status": order.status.upper()` — no null check before chained method call

**Fix:** `"status": (order.status or "").upper()` or add a `NOT NULL` constraint on the `orders.status` column and document it

#### Important (SOFT signals — should fix, not blocking)

**Performance — N+1 query** `app/routes/orders.py:28`

A `session.query(LineItem)` call fires inside the `for order in orders` loop. For 100 orders this makes 101 database round-trips.

**Evidence:** Loop at line 27 iterates `orders`; line 28 executes a query on every iteration.

**Fix:** Fetch all line items in one query before the loop:
```python
order_ids = [o.id for o in orders]
items_by_order = {}
for item in session.query(LineItem).filter(LineItem.order_id.in_(order_ids)).all():
    items_by_order.setdefault(item.order_id, []).append(item)
```
Then replace `items = session.query(...)` in the loop with `items = items_by_order.get(order.id, [])`.
