## Evaluation Criteria (defined before implementation)

Before recommending anything, here are the pass/fail gates this pipeline must meet:

| Criterion | Target |
|---|---|

These are checked in order: retrieval metrics first, then generation. If retrieval fails, we fix it before touching generation.

## Chunking Configuration

Given the structured Confluence content with clear headings:

- **Strategy:** Semantic chunking (split on H1/H2/H3 boundaries) for Confluence pages; paragraph-based chunking for PDFs where structure is detected; 512-token fixed-size with 20% overlap as fallback for PDFs without structure
- **Chunk size:** Variable for semantic (target 300-600 tokens per section); 512 tokens fixed for fallback
- **Overlap:** None for semantic chunks (heading boundaries are clean); 20% (~100 tokens) for fixed-size
- **Boundary rules:** Never split mid-sentence. Prefer section headings as primary boundaries

**Metadata schema (MANDATORY — required for citation and filtered retrieval):**

| Field | Type | Purpose |
|---|---|---|
| source_id | string | Confluence page ID or PDF filename hash |
| source_name | string | Human-readable page title or runbook name |
| chunk_index | integer | Position within source document |
| section_heading | string | Nearest H1/H2/H3 above this chunk |
| created_at | datetime | Indexing timestamp |
| content_hash | string | SHA-256 of chunk text — detects changes on re-index |
| doc_type | string | `confluence_page` or `pdf_runbook` |
| team_owner | string | Extracted from Confluence space metadata where available |

## Embedding Selection

**Decision checkpoint:** Do not select an embedding model without running the 20-query evaluation. Given the $500/month budget, embedding cost at rebuild frequency matters.

**Evaluation method:**
1. Collect 20 representative queries from the support and engineering teams (e.g., "rollback procedure payments service", "authentication microservice owner", "how to restart the billing worker")
2. Manually identify the 3-5 documents that should be retrieved for each query (ground truth)
3. Test two candidates: `text-embedding-3-small` (lower cost, 1536 dimensions) and `text-embedding-3-large` (higher quality, higher cost)
4. Measure Precision@5 and Recall@5 against ground truth
5. Select the model that meets the >= 80% precision target at lowest cost

**Cost consideration at 800MB corpus:** Full re-index at weekly cadence is ~10M tokens/week for the delta. `text-embedding-3-small` at $0.02/1M tokens = ~$0.20/week re-index cost. Well within budget — embedding cost is not the constraint here.

**Placeholder until evaluation runs:** Recommend starting with `text-embedding-3-small`. Upgrade to `text-embedding-3-large` only if the 20-query eval shows precision below 80%.

## Retrieval Configuration

- **Strategy:** Similarity search (cosine) as starting point. Add hybrid search (similarity + BM25) only if eval shows retrieval misses on exact-match queries like error codes or service names
- **Top-K:** 5 (default). Reduces to 3 if latency budget is tight; increases to 10 if faithfulness scores reveal missing context
- **Filters:** `doc_type` filter to let users scope to runbooks only; `team_owner` filter where populated

**Rule:** Start with K=5 similarity search. Only add re-ranking or hybrid search if evaluation shows retrieval is insufficient.

## Freshness Strategy

Given weekly updates (~10-20 documents/week):

- **Rebuild frequency:** Incremental re-index every 24 hours (nightly). Full re-index weekly
- **Change detection:** Content hash comparison — only re-embed chunks where `content_hash` has changed
- **Staleness threshold:** Documents not refreshed within 14 days flagged in retrieval results with a staleness indicator
- **Automation:** Index rebuild must be automated (cron or pipeline trigger). Manual rebuilds will be skipped

---

## Fallback Handling

| Scenario | Handling |
|---|---|
| Generation model unavailable | Queue request, retry after 30s. If still unavailable, return: "AI search is temporarily unavailable. Please use the direct documentation links." |
| Context insufficient (no relevant chunks retrieved) | Return: "I don't have enough information to answer this from the available documentation." with a link to the full docs search |
| Similarity scores all below 0.5 | Treat as zero-context case — do not generate a speculative answer |
| Cost spike (>2x expected per-query cost) | Circuit breaker alert to engineering. Suspend AI search, fall back to keyword search |
