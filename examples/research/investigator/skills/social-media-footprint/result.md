## Notes

The skill's ethics rails are among the strongest in the investigator plugin — the authorisation gate requirement, public-only hard limit, observations-not-conclusions rule, and absence-as-finding principle are all stated explicitly with examples. The structure maps cleanly to the output template, reducing the risk of steps being omitted.

The main gap is the cadence scale: "active / occasional / dormant" are used in Step 1 without defined time thresholds, leaving classification to agent discretion. The test.md defines these categories explicitly (Active = multiple posts/week, Occasional = multiple per month, Sparse = fewer than monthly, Dormant = no activity >6 months) — that vocabulary belongs in the skill itself. A tighter version would also add a `**Purpose:**` line to the gate record template to prompt the model to carry the invoking context into the output, which would fully close the conference-keynote relevance criterion without relying on the Observations section to pick it up opportunistically.
